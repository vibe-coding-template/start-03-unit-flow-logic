# 指令流邏輯 — 單次觸發與重複發送

歡迎來到 **指令流邏輯 (Temporal Logic & Safety)** 單元！

在這個單元中，你將學習如何將「單次的點擊指令」轉化為「連續的動態節奏」。這對於開發高頻控制系統（如賽道競技、機械臂控制或自動化終端）至關重要。你將掌握 `setInterval` 的精準管理、多按鍵狀態聚合，以及最重要的「失效安全 (Fail-safe)」機制。

## 🎯 學習目標
- 掌握 `setInterval` 與 `clearInterval` 的生命週期管理。
- 實作「狀態機」概念處理多按鍵同時輸入的衝突。
- 建立安全防護意識，實作「死人開關 (Dead Man's Switch)」機制。

---

## ✍️ 作業任務 (Assignment Tasks)

請按照以下步驟在 `js/logic.js` 中完成開發：

### 1. 實作指令流控制
- 完成 `startFlow` 與 `stopFlow` 函式。
- **關鍵要點**：確保在連點按鈕或快速按鍵時，不會產生定時器堆疊（Timer Leak）。在啟動新的定時器前，必須確保舊的已被清除。

### 2. 實作多鍵並發邏輯
- 使用 `keysActive` 物件記錄所有按鍵的狀態。
- 確保當同時按下多個按鍵（例如：前進 + 右轉）時，日誌能正確顯示組合後的指令（如：`ArrowUp + ArrowRight`）。

### 3. 實作失效安全機制 (Dead Man's Switch)
- 監聽 `window.blur` (失去焦點) 或 `visibilitychange` (切換分頁) 事件。
- 當使用者離開網頁視窗時，系統必須立即執行 `emergencyStop`，停止所有指令並送出 `STOP` 命令。
